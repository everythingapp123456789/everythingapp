Contributing to The Everything App
Welcome to the resistance. By contributing to this repository, you are not just donating code; you are earning Fractional Ownership in a global, peer-owned ecosystem.

💎 The Ownership Model
We operate on a Proof of Contribution (PoC) system. Every merged Pull Request (PR) is assigned a Complexity Score by the community and maintainers.

Complexity 1-3: Small fixes, documentation, or minor UI tweaks.

Complexity 4-7: New features, API integrations, or bug fixes in the core logic.

Complexity 8-10: Architectural shifts, new modules (e.g., launching the "Food Delivery" shell), or major security audits.

Your Ownership Share = (Your Total Complexity Points / Total Global Points)

🛠 How to Contribute
1. Claim or Propose
Check the Issues tab. If you see a task you want, comment !claim. If you have a new idea for a module (e.g., "Decentralized AWS"), open a new Issue first to discuss the architecture.

2. The Development Flow
Fork the repo and create a branch: git checkout -b feature/your-feature-name.

Build: Keep your code modular. If you're building a new service, it belongs in the /modules folder.

Test: Ensure you don't break the Core Kernel.

3. Submitting a PR
When you submit your PR, you must include the following in the description:

Proposed Complexity Score: (e.g., "I think this is a 5/10")

Wallet Address: Where your future revenue/governance tokens should be recorded.

⚖️ Rules of Engagement
Anti-Fragility: No code is merged without at least two peer reviews.

No Oligarchs: We do not accept "Investment" that grants voting rights. Only participation (code, labor, or service) grants power.

The Kill Switch: If a module is found to be extractive or malicious, the community can vote to "Deprecate" it from the Core Kernel.
